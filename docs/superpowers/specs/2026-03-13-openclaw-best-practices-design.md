# OpenClaw Workspace Best Practices Article Design

## Goal

`#05 AI/#01 Open-Claw` 시리즈의 `04`번 글로, GitHub에서 star를 많이 받은 OpenClaw 관련 자료를 바탕으로 `SOUL.md`와 `MEMORY.md`를 용도별로 어떻게 쓰는지 best practice 3가지를 정리한다.

## Audience

- `03` 글까지 읽고 실제 운용 패턴이 궁금한 독자
- SOUL/MEMORY를 추상적으로 이해했지만 어떤 식으로 써야 할지 막막한 사용자
- GitHub 커뮤니티에서 검증된 패턴을 빠르게 가져오고 싶은 독자

## Structure

1. 왜 이번에는 star 많은 GitHub 자료를 보나
2. 공통 원칙: SOUL/MEMORY/AGENTS의 경계
3. 패턴 1: 개발 보조형
4. 패턴 2: 콘텐츠/리서치형
5. 패턴 3: 개인 비서/운영형
6. 내 OpenClaw에 적용하는 순서

## Constraints

- 스타 수와 날짜를 본문에 명시
- 각 패턴마다 `SOUL.md`, `MEMORY.md`, `하면 안 되는 것`, `바로 쓸 예시` 포함
- 설치/Telegram 글과 톤을 맞추되 과장하지 않기
