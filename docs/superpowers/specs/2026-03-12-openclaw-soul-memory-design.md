# OpenClaw SOUL And MEMORY Article Design

## Goal

`#05 AI/#01 Open-Claw` 시리즈의 `03`번 글로, Telegram 연결 다음 단계에서 `SOUL.md`와 `MEMORY.md`가 각각 무엇을 담당하는지 설명하고, 실제로 어떤 식으로 수정하면 체감이 바뀌는지 실전 예시까지 보여준다.

## Audience

- `00~02` 글까지 읽은 독자
- OpenClaw의 말투, 성격, 기억을 직접 손보고 싶은 입문자
- `SOUL.md`, `MEMORY.md`, `AGENTS.md`의 경계를 헷갈리는 독자

## Structure

1. 왜 이 단계가 중요한가
2. 파일이 어디 있고 언제 읽히는가
3. `SOUL.md`와 `MEMORY.md`의 역할 차이
4. 실제로 어떻게 수정하나
5. 자주 하는 실수
6. 다음 글 연결

## Constraints

- 설치 글보다 짧거나 비슷한 길이
- 개념 설명과 실전 수정 예시를 둘 다 포함
- 로컬 이미지 2개 이상 + Mermaid 1개
- 공식 문서와 커뮤니티 실전 팁을 함께 반영
